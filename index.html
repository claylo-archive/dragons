About

Atlassian's suite of tools can improve a development team's productivity. However, the tools themselves are notoriously difficult to install and configure to work together.

To address this, Atlassian publishes their Atlassian Dragons challenge. After making it most of the way through that challenge, I found it did not solve the installation process in the way I wanted. Atlassian tools are installed over a range of ports, some with different hostnames, and with all credentials hanging bare-assed to the open wire. None of that was okay with me.

So I started over, with the following goals:

  • One URL, one port.
  • SSL wrapped around the whole thing.
  • Leverage the fine Mac OS X server I already had running.
The markdown notes that live in the associated Github repository were created on the fly as I was getting things set up.

The published notes aren't complete, but they get you far enough along that you can apply what you learn in the published steps to the Atlassian Dragons steps to get the rest of the way through the setup.

Please let me know in the comments below if you find this helpful.
